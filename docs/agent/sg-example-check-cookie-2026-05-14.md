# 例題対応チェック結果（2026-05-14）

## 1. 例題の論点

- 問われているテーマ：
  - HTTP Cookie の基本挙動（どこに保存され、どう送受信されるか）
- 正解に必要な知識：
  - Cookie は HTTP ヘッダで扱われる状態管理の仕組みである
  - ブラウザは保存済み Cookie を `Cookie` ヘッダでサーバに送る
- 誤答選択肢を切るポイント：
  - 「Webサーバだけに保存される」は誤り（保存先はブラウザ）
  - 「ブラウザが全て暗号化して送る」は誤り（暗号化は HTTPS/TLS の役割）
  - 「有効期限はクライアントが設定する」は誤り（通常はサーバが `Set-Cookie` 属性で指定）
- 混同しやすい用語：
  - `Set-Cookie` と `Cookie`
  - Cookie と HTTPS
  - Cookie本体の保存先（ブラウザ）とセッション実体の保存先（サーバ側）

## 4. 理由

今回の例題は「Cookie情報がHTTPヘッダの一部として送信されるか」を問う基本問題で、
`pages/sg/sg_cookie.md` に以下が明示されているため、正解選択・誤答除去の両方に対応できる。

- `Set-Cookie`（サーバ→ブラウザ）と `Cookie`（ブラウザ→サーバ）の方向
- Cookie保存場所（ブラウザ）
- 暗号化主体はCookieではなくHTTPS
- 有効期限は `Expires/Max-Age` をサーバ側で指定する考え方

このため、SG試験の「似ているが違う選択肢」を切るための判断軸は既存記事で満たせている。
