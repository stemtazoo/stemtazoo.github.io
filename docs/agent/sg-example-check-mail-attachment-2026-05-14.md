# SG例題対応チェック（メール添付ファイル対策）

- 実施日: 2026-05-14
- 対象例題: 電子メール経由のウイルス被害回避（a,b,cの正誤組合せ）

## 例題の論点

- 信頼できる送信者でも添付ファイルは安全とは限らない（a）
- 非実行形式（画像・音声・テキストに見える拡張子）でも偽装・悪用があり得る（b）
- 不審メールをプレビューで安全確認して閲覧する、は不適切（c）
- 選択肢の切り分け軸は「開く前の予防行動」と「誤った安心材料（送信者名・見た目・プレビュー）」

## 関連既存記事の確認結果

- `pages/sg/suspicious-attachments.md`
  - 主論点（開かない判断、送信元・文脈確認、見た目偽装）に直接対応。
  - ただし「プレビューで安全確認してから開く」誤解を明示的に切る文が弱い。
- `pages/sg/targeted-attack-email.md`
  - 不審メールは開かず報告という行動が整理されており、cを切る補助になる。
- `pages/sg/macro-virus.md` / `pages/sg/malware.md`
  - 添付経由感染の背景知識として有効。

## 判定

- 判定: **既存記事に追記推奨**
- 優先度: **中**

## 推奨追記（最小）

追記先: `pages/sg/suspicious-attachments.md`

追加候補見出し（既存「よくある誤解・混同」配下）:

- 「誤解：プレビューで確認すれば安全」

追記要点:

- プレビューは安全確認手段ではない。
- 不審メールは「開いて確認」ではなく「開かず報告・削除（組織手順に従う）」で判断。
- SG試験では「安全性を確認してから閲覧する」という文言が出たら注意し、
  「不審なものは開かない」が正答軸になりやすい。

## 補足（学習導線）

- 用語比較は既存導線で概ね足りる。
  - 不審な添付ファイル（開封判断）
  - 標的型攻撃メール（報告行動）
  - フィッシング（入力詐取）
