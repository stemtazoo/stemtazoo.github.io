## 1. 例題の論点

- 問われているテーマ:
  - NIDS（ネットワーク型IDS）の目的と監視対象
- 正解に必要な知識:
  - IDSは「検知・通知」が中心で、遮断は主目的ではない
  - NIDSはネットワーク上の通信を監視する
- 誤答選択肢を切るポイント:
  - 「実際に攻撃して侵入可能性を確かめる」はペネトレーションテスト
  - 「損害の大きさを判定する」はリスクアセスメント
  - 「サーバ上のファイル改ざん判定」はHIDS寄り
- 混同しやすい用語:
  - NIDS / HIDS
  - IDS / IPS
  - IDS / ペネトレーションテスト
  - IDS / リスクアセスメント

## 4. 理由

`pages/sg/ids.md` だけでも正答（ア）を選ぶための知識は十分あります。

一方で、今回の誤答（イ・ウ・エ）は「似たセキュリティ活動を混同させる」典型で、試験本番では次のような切り分けを瞬時にできる形にしておくと得点安定につながります。

- 侵入して確かめる: ペネトレーションテスト
- 損害規模を見積もる: リスクアセスメント
- ホスト内改ざんを監視: HIDS
- ネットワーク通信を監視し通知: NIDS

この4点を、IDS記事内の「確認問題」または「よくある誤解・混同」に直結表として追加すると、同種問題への再現性が上がります。

## 5. 具体的な対応案

### 既存記事に追記する場合

- 追記先の記事:
  - `pages/sg/ids.md`
- 追記する見出し:
  - `## よくある誤解・混同` 内に「NIDSの目的と他活動の切り分け（4択頻出）」
  - または `## 確認問題（SG試験対策）`（既存ルールに従って追加）
- 追記内容の要約:
  - NIDSの目的を、ペネトレーションテスト・リスクアセスメント・HIDSと並べて比較する
- 追記すべき本文案:

```md
### NIDSの目的と他活動の切り分け（4択頻出）

SG試験では、NIDSの説明に似せて別活動を混ぜる問題がよく出ます。

- ネットワーク上の不正通信を検知し、管理者に通知する → NIDS
- 実際に侵入できるかを攻撃者視点で試す → ペネトレーションテスト
- 攻撃時の影響度や損害規模を見積もる → リスクアセスメント
- サーバ内のログやファイル改ざんを監視する → HIDS

👉 判断ポイント
「何を監視する仕組みか（通信か、ホスト内部か）」と
「監視ではなく評価・実証の活動ではないか」を先に確認する。
```
