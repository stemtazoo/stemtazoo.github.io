# SG例題対応チェック（保証型監査と助言型監査）

- 作成日: 2026-05-13
- 対象: 情報セキュリティ監査における保証型監査と助言型監査の実施に関する4択問題
- 判定: 既存記事に追記推奨（実施済み）

## 1. 例題の論点

- 問われているテーマ:
  - 保証型監査と助言型監査の目的・関係・選択主体
- 正解に必要な知識:
  - 不特定多数の利害関係者が関与する場面で、保証型監査の定期実施と結果開示が有用
- 誤答選択肢を切るポイント:
  - 保証型→助言型の固定順序は不要（むしろ助言型→保証型の流れが自然な場合もある）
  - 保証型と助言型は完全排他ではない
  - どちらの型かは監査人が単独で決めるのではなく、監査目的・依頼者/被監査側ニーズで決まる
- 混同しやすい用語:
  - 保証型監査 / 助言型監査
  - 排他的選択 / 時期を分けた使い分け
  - 監査人の裁量 / 監査目的の事前合意

## 3. 判定

既存記事に追記推奨（対応済み）

## 4. 理由

既存の保証型/助言型記事で主要論点は押さえられていましたが、今回のように「固定順序」「排他性」「決定主体」を同時に問う設問では、
「助言型で改善→保証型で適切性評価」という時系列の整理があると誤答を切りやすくなります。

## 5. 具体的な対応案

### 既存記事に追記する場合

- 追記先の記事:
  - `pages/sg/assurance-advisory-audit.md`
- 追記する見出し:
  - `## どんな場面で使う？` の末尾説明を拡張
  - `## よくある誤解・混同` の誤解3を補強
- 追記内容の要約:
  - 助言型→保証型への移行はあり得ること
  - ただし同一時点で目的を混ぜすぎるのは不適切であること
- 追記した本文要点:
  - 体制未整備期は助言型、成熟後は保証型
  - 非排他だが、監査計画で目的分離が必要
