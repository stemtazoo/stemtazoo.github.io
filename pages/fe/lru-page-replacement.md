## まず結論

LRU方式とは、ページ置換が必要になったときに、**最後に参照されてから最も時間がたっているページを追い出す方式**です。

LRUは、Least Recently Used の略です。

基本情報技術者試験では、LRU方式は **「最近使っていないページ」を置換対象にする** と覚えると判断しやすいです。

計算問題では、ページ枠の中身だけでなく、**各ページが最後にいつ参照されたか** を追うことが大切です。

## 直感的な説明

LRU方式は、机の上の資料整理にたとえると分かりやすいです。

机の上には、資料を4冊しか置けないとします。

新しい資料を置きたいのに机がいっぱいなら、どれか1冊を片付ける必要があります。

このとき、LRU方式では、**一番長い間使っていない資料** を片付けます。

```text
最近使った資料 → また使うかもしれないので残す
長く使っていない資料 → 使う可能性が低そうなので追い出す
```

ページ置換でも同じです。

最近参照されたページは、近いうちにまた参照される可能性があると考え、長く参照されていないページを置換対象にします。

## 定義・仕組み

仮想記憶では、プログラムが使うページの一部だけを主記憶に置きます。

主記憶のページ枠がいっぱいの状態で、新しいページを読み込む必要があると、どれかのページを追い出さなければなりません。

このとき、どのページを追い出すかを決める方法が、ページ置換アルゴリズムです。

| 用語 | 意味 |
|---|---|
| ページ | 仮想記憶で扱う固定サイズの単位 |
| ページ枠 | 主記憶上でページを入れる場所 |
| ページフォールト | 必要なページが主記憶にない状態 |
| ページ置換 | 主記憶のページを入れ替えること |
| LRU方式 | 最後の参照時刻が最も古いページを置き換える方式 |

LRU方式では、ページが参照されるたびに、そのページを「最近使った」として扱います。

ページ枠がいっぱいで新しいページを入れるときは、枠内にあるページのうち、**最後に参照された時刻が最も古いページ** を置き換えます。

このテーマは、基本情報技術者試験の「コンピュータシステム」や「オペレーティングシステム」と関係する内容です。公式の出題範囲やシラバスは、[IPA：基本情報技術者試験](https://www.ipa.go.jp/shiken/kubun/fe.html) から確認できます。

## 科目Aでどう出る？

科目Aでは、LRU方式の説明や、ページ参照列を使った計算問題として出題されやすいです。

判断するときは、次の表現を見ます。

| 表現 | 判断 |
|---|---|
| 最後に参照されてから最も時間がたったページ | LRU方式 |
| 最近使われていないページを置き換える | LRU方式 |
| 最も古く主記憶に入ったページを置き換える | FIFO方式 |
| これから最も長く使われないページを置き換える | OPT方式 |
| 使用頻度が最も低いページを置き換える | LFU方式 |

LRUで特に大切なのは、**入った順番ではなく、参照された順番を見る** ことです。

例えば、ページ枠に `1, 2, 3, 4` が入っていて、その後にページ2を参照したとします。

この場合、ページ2は「最近使ったページ」になります。

そのため、次に置換が必要になっても、ページ2を古いものとして扱ってはいけません。

## 科目Bでどう使う？

科目Bでは、ページ参照列を順に追って、ページ枠の中身を更新する問題でLRUの考え方が役立ちます。

基本手順は次のとおりです。

1. ページ参照列を左から順に見る
2. 参照ページがページ枠にあれば、置換はしない
3. ただし、そのページは「最近使った」として更新する
4. 参照ページがページ枠になければ、ページフォールトになる
5. 空き枠があれば、そのまま入れる
6. 空き枠がなければ、最後の参照が最も古いページを置き換える

例えば、ページ枠が3つで、次の順に参照するとします。

```text
1, 2, 3, 2, 4
```

途中の状態を簡単に追うと、次のようになります。

| 参照 | ページ枠 | 考え方 |
|---|---|---|
| 1 | 1 | 空き枠に入れる |
| 2 | 1, 2 | 空き枠に入れる |
| 3 | 1, 2, 3 | 空き枠に入れる |
| 2 | 1, 2, 3 | 2を最近使った扱いにする |
| 4 | 4, 2, 3 | 最後の参照が最も古い1を置き換える |

ここで、ページ2を参照した後に、ページ2を古いままと考えると間違えます。

LRU方式では、**ヒットしたページも参照時刻を更新する** のがポイントです。

## よくある誤解・混同

LRU方式でよくある誤解は、**ページ枠に入った順番だけで判断してしまうこと** です。

| 誤解 | 正しい理解 |
|---|---|
| 最初に入ったページを追い出す | それはFIFO方式 |
| ページ枠にあるページを見たら何もしない | 置換はしないが、参照時刻は更新する |
| 数字が小さいページを追い出す | ページ番号の大小は関係ない |
| 直前に使ったページを追い出す | LRUは最も最近使っていないページを追い出す |
| 最も多く使われたページを残す | LRUは頻度ではなく最後の参照時刻を見る |

FIFO方式との違いは、特に重要です。

| 方式 | 見るもの | 置き換えるページ |
|---|---|---|
| FIFO | 主記憶に入った順番 | 最初に入ったページ |
| LRU | 最後に参照された時刻 | 最後の参照が最も古いページ |

試験では、次のように覚えると切り分けやすいです。

```text
FIFO：先に入ったものを出す
LRU：最近使っていないものを出す
```

## まとめ（試験直前用）

- LRU方式は、最後に参照されてから最も時間がたったページを置き換える
- LRUは Least Recently Used の略
- ページ番号の大小では判断しない
- ページ枠にヒットした場合も、参照時刻は更新する
- FIFOは入った順番、LRUは最後に使った時刻で判断する
- 計算問題では、参照列を左から順に追う
