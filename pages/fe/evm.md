## まず結論

EVMとは、**プロジェクトの作業を金額の価値に置き換えて、進捗とコストを定量的に管理する手法**です。

EVMは、Earned Value Management の略です。

基本情報技術者試験では、まず次の2つを切り分けます。

| 見る式 | 何を見る？ | 判断 |
|---|---|---|
| EV − PV | スケジュール・進捗 | 正なら進んでいる、負なら遅れている |
| EV − AC | コスト | 正なら予算内、負ならコスト超過 |

今回のように **EV − PV が負** なら、次のように判断します。

```text
EV − PV < 0
→ EV < PV
→ 実際に完了した作業量が、計画より少ない
→ 進捗が計画より遅れている
```

つまり、**EV − PV が負なら、現時点の進捗遅れ** です。

## 直感的な説明

EVMは、プロジェクトの進み具合を「作業量」ではなく、**お金の価値**で見る方法です。

例えば、計画ではここまでに100万円分の作業が終わっているはずだったとします。

でも、実際には80万円分の作業しか終わっていなければ、進捗は遅れています。

```text
PV：計画では100万円分進んでいるはず
EV：実際には80万円分しか終わっていない

EV − PV = 80 − 100 = -20
→ 進捗遅れ
```

逆に、実際にかかった費用を見るときは、ACを使います。

```text
EV：80万円分の成果
AC：90万円かかった

EV − AC = 80 − 90 = -10
→ コスト超過
```

EVMでは、**進捗を見る式** と **コストを見る式** を混ぜないことが大切です。

## 定義・仕組み

EVMでは、主に次の3つの値を使います。

| 指標 | 英語 | 意味 | ざっくり |
|---|---|---|---|
| PV | Planned Value | ある時点までに計画されていた作業の予算価値 | 計画ではどれだけ進む予定か |
| EV | Earned Value | ある時点までに完了した作業の予算価値 | 実際にどれだけ成果が出たか |
| AC | Actual Cost | ある時点までに完了した作業に実際にかかった費用 | 実際にいくら使ったか |

ここから、スケジュールやコストの状態を見ます。

| 指標 | 式 | 意味 |
|---|---|---|
| SV | EV − PV | スケジュール差異 |
| CV | EV − AC | コスト差異 |
| SPI | EV ÷ PV | スケジュール効率指数 |
| CPI | EV ÷ AC | コスト効率指数 |

FE試験では、SVやCVという名前が出なくても、**EV − PV** や **EV − AC** の意味を読めれば十分です。

このテーマは、基本情報技術者試験の「プロジェクトマネジメント」と関係する内容です。公式の出題範囲やシラバスは、[IPA：基本情報技術者試験](https://www.ipa.go.jp/shiken/kubun/fe.html) から確認できます。

## 科目Aでどう出る？

科目Aでは、EVMの式から状態を選ぶ問題が出やすいです。

まずは、何を比較しているかを見ます。

| 比較 | 見ているもの | 判断基準 |
|---|---|---|
| EV − PV | 進捗・スケジュール | 正なら進んでいる、負なら遅れている |
| EV − AC | コスト | 正なら予算内、負ならコスト超過 |
| EV ÷ PV | スケジュール効率 | 1より大きいなら良い、1より小さいなら悪い |
| EV ÷ AC | コスト効率 | 1より大きいなら良い、1より小さいなら悪い |

今回のような選択肢では、次のように切ります。

| 選択肢の内容 | 判断 |
|---|---|
| スケジュール効率が計画より良い | EV − PV が負なので違う。進捗は悪い方向 |
| プロジェクトの完了が計画より遅くなる | 現時点の進捗遅れは分かるが、完了時点までは断定できない |
| プロジェクトの進捗が計画より遅れている | 正しい |
| プロジェクトの進捗が計画より進んでいる | EV − PV が正の場合の説明 |

特に、**現時点の進捗遅れ** と **プロジェクト完了の遅れ** は分けます。

EVMのその時点の値から分かるのは、基本的には **その時点での状態** です。

問題文に将来予測の条件がないのに、完了が遅くなると断定する選択肢は慎重に見ます。

## 科目Bでどう使う？

科目Bでは、EVMの式そのものよりも、プロジェクトの状況を読むために使います。

例えば、次のように整理します。

```text
EV < PV
→ 予定より成果が少ない
→ 進捗遅れ

EV > PV
→ 予定より成果が多い
→ 進捗前倒し

EV < AC
→ 成果より費用が多い
→ コスト超過

EV > AC
→ 成果より費用が少ない
→ コスト効率がよい
```

科目Bでは、文章量が多くなるので、まず **EVは成果、PVは計画、ACは実費** と置き換えると読みやすくなります。

```text
EV：できた分
PV：やる予定だった分
AC：実際に使ったお金
```

この3つに置き換えてから、差や比率を見ます。

## よくある誤解・混同

EVMでは、次の混同がよく起こります。

| 混同 | 正しい切り分け |
|---|---|
| EV − PV をコストの式だと思う | EV − PV はスケジュール・進捗を見る |
| EV − AC を進捗の式だと思う | EV − AC はコストを見る |
| EV − PV が負なら、必ずプロジェクト完了が遅れる | 分かるのは現時点の進捗遅れ。完了遅れまでは条件次第 |
| SPIが1未満なのに良いと判断する | SPI = EV ÷ PV。1未満なら進捗効率が悪い |
| CPIが1未満なのに良いと判断する | CPI = EV ÷ AC。1未満ならコスト効率が悪い |

特に、EVとPVの順番を逆にしないようにします。

```text
EV − PV
→ できた分 − 予定していた分
```

だから、負なら「できた分が予定より少ない」と読めます。

## まとめ（試験直前用）

- EVMは、作業を金額価値に置き換えて進捗とコストを管理する手法
- PVは計画、EVは成果、ACは実費
- EV − PV はスケジュール・進捗を見る
- EV − PV が負なら、進捗が計画より遅れている
- EV − AC はコストを見る
- EV − AC が負なら、コスト超過
- SPI = EV ÷ PV、1未満ならスケジュール効率が悪い
- CPI = EV ÷ AC、1未満ならコスト効率が悪い
- 現時点の進捗遅れと、プロジェクト完了の遅れは分けて考える
