## まず結論

**プログラムカウンタは、次に実行する命令が格納されているメモリアドレスを保持するレジスタです。**

基本情報技術者試験では、CPU内部のレジスタを「何を保持するか」で切り分けます。

| レジスタ | 主に保持するもの |
|---|---|
| プログラムカウンタ | 次に実行する命令のアドレス |
| 命令レジスタ | 現在読み出した命令 |
| 汎用レジスタ | 演算に使うデータや途中結果 |
| アキュムレータ | 演算結果 |
| フラグレジスタ | 演算結果の状態 |

試験では、次の一文を判断軸にします。

> **次の命令の場所はプログラムカウンタ、現在の命令は命令レジスタ。**

## 直感的な説明

プログラムカウンタは、本の**しおり**のようなものです。

本を読んでいるとき、しおりは「次に読むページ」を示します。CPUでも同じように、プログラムカウンタが「次に読み出す命令の場所」を示します。

```text
プログラムカウンタ
↓
次の命令のアドレスを示す
↓
メモリから命令を読み出す
```

例えば、プログラムカウンタに `100` が入っている場合、CPUはメモリの100番地から命令を読み出します。

```text
プログラムカウンタ：100
↓
メモリ100番地の命令を読み出す
↓
次の命令を指す値へ更新する
```

通常は次の命令を指すように進みますが、分岐命令やジャンプ命令では、別のアドレスが設定されます。

## 定義・仕組み

CPUは、命令を次の流れで実行します。

```text
1. 次の命令の場所を確認する
2. メモリから命令を読み出す
3. 命令を保持する
4. 命令を解読する
5. 必要なデータを読み込む
6. 演算する
7. 結果や状態を保存する
```

この流れに各レジスタを対応させると、役割が分かりやすくなります。

### プログラムカウンタ

次に実行する命令の**アドレス**を保持します。

```text
次はどこを読むか
→ プログラムカウンタ
```

命令を読み出した後は、通常、次の命令を指すように値が更新されます。

分岐命令の場合は、分岐先のアドレスが設定されます。

### 命令レジスタ

メモリから読み出した、現在実行する**命令そのもの**を保持します。

```text
読み出した命令
→ 命令レジスタ
```

CPUは命令レジスタの内容を解読し、どの処理を行うか決めます。

### 汎用レジスタ

演算に使うデータ、途中結果、アドレスなどを一時的に保持します。

```text
計算に使う値
途中の結果
一時的なアドレス
→ 汎用レジスタ
```

「汎用」という名前のとおり、用途が一つに固定されていません。

### アキュムレータ

主に演算結果を保持するレジスタです。

```text
演算結果
→ アキュムレータ
```

古典的なCPUでは、演算結果をアキュムレータへ集めて処理します。現在のCPUでは、汎用レジスタが同様の役割を兼ねることもあります。

### フラグレジスタ

演算結果そのものではなく、演算結果の**状態**を保持します。

例えば、次の状態です。

- 結果が0か
- 桁上がりが発生したか
- 結果が負か
- オーバーフローしたか

```text
演算を実行
↓
結果の状態をフラグに記録
↓
条件分岐で利用
```

## 科目Aでどう出る？

科目Aでは、レジスタの説明を見て名称を選ぶ問題がよく出ます。

判断するときは、選択肢に含まれる中心語を確認します。

```text
次の命令のアドレス
→ プログラムカウンタ

現在の命令を保持してデコード
→ 命令レジスタ

演算用データや途中結果
→ 汎用レジスタ

演算結果
→ アキュムレータ

演算結果の状態
→ フラグレジスタ
```

### 典型的な切り分け

| 選択肢の表現 | 対応するレジスタ |
|---|---|
| 次の命令が格納されたアドレスを保持 | プログラムカウンタ |
| メモリから読み出した命令を保持 | 命令レジスタ |
| 演算のためにデータを一時保持 | 汎用レジスタ |
| 演算結果を保持 | アキュムレータ |
| ゼロ、桁上がりなどの状態を保持 | フラグレジスタ |

## どんな場面で使う？

### プログラムカウンタ

命令の実行順序を管理するときに使います。

通常は順番に命令を実行しますが、条件分岐やジャンプが起きると、プログラムカウンタの値を書き換えます。

```text
条件を満たす
↓
分岐先アドレスをプログラムカウンタへ設定
↓
分岐先の命令を実行
```

### 命令レジスタ

現在実行中の命令を解読するときに使います。

```text
命令を読み出す
↓
命令レジスタに保持
↓
制御装置が命令を解読
```

### フラグレジスタ

条件付き分岐で使います。

例えば、比較結果が等しい場合だけ分岐する処理では、ゼロフラグなどを確認します。

```text
比較を行う
↓
結果の状態をフラグへ記録
↓
フラグを見て分岐する
```

## よくある誤解・混同

### プログラムカウンタと命令レジスタ

最も混同しやすい組合せです。

```text
プログラムカウンタ
→ 次の命令の住所

命令レジスタ
→ 現在の命令の内容
```

例えば、100番地に命令がある場合は次のようになります。

```text
プログラムカウンタ
→ 100

命令レジスタ
→ 100番地から読み出した命令
```

> **住所を持つのがプログラムカウンタ、内容を持つのが命令レジスタ。**

### 汎用レジスタとアキュムレータ

どちらも演算に使われるため混同しやすいです。

```text
汎用レジスタ
→ データや途中結果を幅広く保持

アキュムレータ
→ 主に演算結果を保持
```

試験では、役割を分けて考えます。

### フラグレジスタは演算結果を保持する？

フラグレジスタが保持するのは、演算結果そのものではありません。

```text
演算結果そのもの
→ 汎用レジスタやアキュムレータ

演算結果の状態
→ フラグレジスタ
```

「結果が0か」「桁上がりがあるか」などを保持します。

### プログラムカウンタは命令数を数える？

名称に「カウンタ」とありますが、実行した命令数を数えるものではありません。

保持するのは、次の命令がある**アドレス**です。

```text
実行回数を数える
→ 誤り

次の命令のアドレスを保持
→ 正しい
```

## まとめ（試験直前用）

- **次の命令のアドレス**を保持するのがプログラムカウンタ
- **現在の命令そのもの**を保持するのが命令レジスタ
- **演算用データや途中結果**を保持するのが汎用レジスタ
- **演算結果**を保持するのがアキュムレータ
- **演算結果の状態**を保持するのがフラグレジスタ

試験直前には、次の順で思い出します。

```text
次はどこ？
→ プログラムカウンタ

今の命令は何？
→ 命令レジスタ

計算に使う値は？
→ 汎用レジスタ

計算結果は？
→ アキュムレータ

結果の状態は？
→ フラグレジスタ
```

> **次の場所はプログラムカウンタ、今の命令は命令レジスタ。**
