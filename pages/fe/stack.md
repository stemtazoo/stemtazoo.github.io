## まず結論

スタックとは、**最後に入れたデータを、最初に取り出す** データ構造です。

基本情報技術者試験では、  
**スタック = 後入れ先出し（LIFO）**  
と判断できることが大切です。

特に科目Bでは、`push` と `pop` の順番を追いかけて、  
「最後に入った値がどれか」を読む力が問われます。

## 直感的な説明

スタックは、**積み重ねた皿** をイメージすると分かりやすいです。

皿を重ねるときは、上にどんどん置きます。  
取り出すときは、一番上の皿から取ります。

つまり、

```text
入れる：上に積む
出す　：一番上から取る
```

という動きです。

後から置いた皿ほど、先に取り出されます。

この考え方が、スタックの基本です。

英語では **Last In, First Out** といい、  
よく **LIFO** と略されます。

## 定義・仕組み

スタックは、データを一時的にためておき、  
最後に入れたデータから順番に取り出す仕組みです。

代表的な操作は、次の2つです。

| 操作 | 意味 |
|---|---|
| push | データをスタックに入れる |
| pop | スタックからデータを取り出す |

たとえば、次の順番でデータを入れたとします。

```text
push(10)
push(20)
push(30)
```

このとき、スタックの中は次のようなイメージです。

```text
上
30  ← 最後に入れた
20
10
下
```

ここで `pop` すると、取り出されるのは `30` です。

```text
pop() → 30
```

もう一度 `pop` すると、次は `20` が取り出されます。

```text
pop() → 20
```

このように、スタックでは  
**最後に入れたものから取り出す**  
という順番になります。

スタックは、基本情報技術者試験の「アルゴリズムとプログラミング」や「データ構造」と関係が深いテーマです。公式の出題範囲は、[IPA：基本情報技術者試験](https://www.ipa.go.jp/shiken/kubun/fe.html) から確認できるシラバスで確認できます。

## 科目Aでどう出る？

科目Aでは、スタックの基本性質を問う問題が出やすいです。

特に見るべきポイントは、次の3つです。

```text
1. 後入れ先出し
2. LIFO
3. push と pop
```

たとえば、選択肢に次のような説明があれば、スタックの説明として適切です。

```text
最後に格納したデータを、最初に取り出す。
```

反対に、次のような説明はスタックではありません。

```text
最初に格納したデータを、最初に取り出す。
```

これは **キュー** の説明です。

スタックとキューは、基本情報技術者試験で混同しやすい代表例です。

| データ構造 | 取り出し方 | 覚え方 |
|---|---|---|
| スタック | 後入れ先出し | 積んだ皿 |
| キュー | 先入れ先出し | 行列 |

科目Aでは、  
**LIFOならスタック、FIFOならキュー**  
と切り分けます。

## 科目Bでどう使う？

科目Bでは、スタックという言葉がそのまま出るとは限りません。

次のような処理を読んで、  
「これはスタックの動きだ」と気づくことが大切です。

```text
データを配列の末尾に入れる
取り出すときは、末尾から取り出す
```

これはスタックの動きです。

科目Bでよくある読み方は、次の流れです。

```text
1. push された値を順番にメモする
2. pop が出たら、一番最後に入れた値を取り出す
3. 取り出した値をスタックから消す
4. 次の処理へ進む
```

たとえば、次の処理を考えます。

```text
push(A)
push(B)
pop()
push(C)
pop()
pop()
```

順番に追うと、こうなります。

| 処理 | スタックの状態 | 取り出される値 |
|---|---|---|
| push(A) | A | - |
| push(B) | A, B | - |
| pop() | A | B |
| push(C) | A, C | - |
| pop() | A | C |
| pop() | 空 | A |

取り出される順番は、次のとおりです。

```text
B → C → A
```

ここで大切なのは、  
**入れた順番ではなく、最後に残っている一番上を見ること** です。

科目Bでは、頭の中だけで追うと間違えやすいです。  
小さな表を書いて、スタックの状態を更新していくと安全です。

## よくある誤解・混同

### 誤解1：入れた順番に取り出すと思ってしまう

スタックは、入れた順番には取り出しません。

```text
push(1)
push(2)
push(3)
```

このあとに取り出すと、順番は次のようになります。

```text
3 → 2 → 1
```

**1 → 2 → 3** ではありません。

### 誤解2：キューと同じだと思ってしまう

スタックとキューは、どちらもデータを一時的にためる仕組みです。

でも、取り出す順番が違います。

| 用語 | 取り出すデータ |
|---|---|
| スタック | 最後に入れたデータ |
| キュー | 最初に入れたデータ |

試験では、  
**「最後」か「最初」か**  
を必ず見ます。

### 誤解3：push と pop の意味を逆に覚える

`push` は入れる操作です。  
`pop` は取り出す操作です。

英語のイメージでは、

```text
push：押し込む
pop：ポンと取り出す
```

くらいで覚えると十分です。

### 誤解4：スタックの中身を消さずに読み続ける

`pop` した値は、スタックから取り出されます。

つまり、取り出したあとは、  
その値はスタックの中には残りません。

科目Bでトレースするときは、  
`pop` のたびに状態を更新することが大切です。

## まとめ（試験直前用）

- スタックは **後入れ先出し（LIFO）** のデータ構造
- `push` は入れる、`pop` は取り出す
- 最後に入れたデータから取り出す
- キューは **先入れ先出し（FIFO）** なので混同しない
- 科目Bでは、`push` と `pop` のたびに状態を書き換えて追う
