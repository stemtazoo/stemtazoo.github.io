## まず結論

状態遷移図とは、**システムが今どの状態にあり、イベントによって次にどの状態へ移るか**を表す図です。

基本情報技術者試験では、状態遷移図は **状態が変わりながら動くシステム** を設計・整理するときに使う、と押さえると分かりやすいです。

単なる集計処理や計算処理ではなく、**現在の状態によって次の動きが変わるか**を見るのが判断ポイントです。

## 直感的な説明

状態遷移図は、システムの **状態の地図** のようなものです。

例えば、自動販売機を考えると、次のように状態が変わります。

```text
待機中
  ↓ お金を入れる
商品選択待ち
  ↓ ボタンを押す
商品払い出し中
  ↓ 払い出し完了
待機中
```

ここで大切なのは、処理がただ順番に進むだけではないことです。

同じ「ボタンを押す」でも、まだお金が足りない状態なら商品は出ません。  
つまり、**今の状態によって、同じ操作でも次の動きが変わる**ことがあります。

このようなシステムを整理するのに、状態遷移図が向いています。

## 定義・仕組み

状態遷移図では、主に次の3つを見ます。

| 要素 | 意味 |
|---|---|
| 状態 | システムが今どういう状況にあるか |
| イベント | 状態が変わるきっかけ |
| 遷移 | ある状態から別の状態へ移ること |

イメージとしては、次のように読みます。

```text
状態A --イベント--> 状態B
```

例えば、ログイン機能なら次のように考えられます。

```text
未ログイン --ログイン成功--> ログイン中
ログイン中 --ログアウト--> 未ログイン
未ログイン --ログイン失敗--> 未ログイン
```

ポイントは、**データの流れ**よりも、**状態の変化**を表すことです。

このテーマは、基本情報技術者試験の「ソフトウェア開発」や「システム設計」と関係する内容です。公式の出題範囲やシラバスは、[IPA：基本情報技術者試験](https://www.ipa.go.jp/shiken/kubun/fe.html) から確認できます。

## 科目Aでどう出る？

科目Aでは、状態遷移図を使うのに適したシステムを選ぶ問題が出やすいです。

判断するときは、次のように見ます。

| 問題文の特徴 | 状態遷移図に向く？ |
|---|---|
| センサー情報によって制御状態が変わる | 向く |
| 画面や処理状態がイベントで変わる | 向く |
| 通信や処理の待ち状態がある | 向く |
| 月末にデータをまとめて集計するだけ | 向きにくい |
| 入力値から料金を計算するだけ | 向きにくい |
| 日次レポートを出力するだけ | 向きにくい |

選択肢では、**状態が変わるシステムかどうか**を見ます。

```text
状態がある
イベントで変わる
次の動きが現在の状態に依存する
```

この3つが見えたら、状態遷移図を疑うとよいです。

逆に、単なる集計、計算、帳票出力であれば、状態遷移図が最適とは限りません。

## 科目Bでどう使う？

科目Bでは、状態遷移図そのものを描かせるというより、問題文の中で **今どの状態にいるか** を追う力が役立ちます。

例えば、次のような処理を読むときです。

```text
待機中
エラー検出
復旧処理中
再試行中
正常稼働中
停止中
```

このような問題では、処理の流れだけでなく、状態を追います。

見るポイントは次の3つです。

```text
1. 今の状態はどこか
2. どのイベントが起きたか
3. 次にどの状態へ移るか
```

科目Bでは、条件分岐やイベントが増えると、読み間違いが起きやすくなります。

そのため、状態遷移図の考え方を使って、**状態を1つずつ追う**と整理しやすくなります。

## よくある誤解・混同

状態遷移図でよくある誤解は、**処理の順番を表す図なら何でも状態遷移図だと思ってしまうこと**です。

| 誤解 | 正しい理解 |
|---|---|
| 手順を並べた図は全部、状態遷移図 | 状態とイベントによる変化を表す図 |
| 集計処理に状態遷移図が最適 | 集計だけなら状態の変化は中心ではない |
| 料金計算に状態遷移図が最適 | 入力値から計算するだけなら、計算式や処理手順の方が自然 |
| フローチャートと同じ | フローチャートは処理手順、状態遷移図は状態の変化に注目する |
| データフロー図と同じ | データフロー図はデータの流れ、状態遷移図は状態の変化に注目する |

特に混同しやすいのは、フローチャートです。

```text
フローチャート
→ 処理の順番を見る

状態遷移図
→ 状態がどう変わるかを見る
```

この違いを押さえると、選択肢を切りやすくなります。

## まとめ（試験直前用）

- 状態遷移図は、状態とイベントによる変化を表す図
- 「今の状態」によって次の動きが変わるシステムに向く
- センサー制御、画面遷移、待ち状態のある処理と相性がよい
- 単なる集計、帳票出力、計算処理は状態遷移図が最適とは限らない
- フローチャートは処理手順、状態遷移図は状態変化を見る
