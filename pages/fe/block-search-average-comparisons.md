## まず結論

ブロック探索では、平均比較回数を公式として丸暗記する必要はありません。

次の2段階に分ければ、その場で式を作れます。

```text
1. 目的のデータがあるブロックを探す
2. 見つけたブロックの中を探す
```

データが全部で `n個`、1ブロックが `m個`なら、平均比較回数は次の形になります。

\[
\frac{n}{2m}+\frac{m}{2}
\]

大切なのは式そのものではなく、

> **外側を半分、内側も半分探す**

という考え方です。

## 直感的な説明

辞書を何冊かの束に分けて探す場面を考えます。

最初に、各束の最後のページだけを見て、目的の言葉がどの束にあるかを探します。

次に、見つけた束の中を先頭から順に探します。

```text
束を探す
→ 束の中を探す
```

このように、探索を2段階に分けるのがブロック探索です。

## 定義・仕組み

昇順に並んだ `n個` のデータを、1ブロック `m個` ずつに分けます。

ブロック数は、

\[
\frac{n}{m}
\]

個です。

各ブロックの最後尾のデータだけを並べて線形探索し、目的のデータが含まれるブロックを特定します。

その後、見つけたブロック内をもう一度線形探索します。

```text
第1段階：ブロック末尾を比較する
第2段階：ブロック内を比較する
```

## 科目Aでどう出る？

科目Aでは、平均比較回数の式を選ぶ問題として出やすいです。

公式を覚えるより、各段階の探索対象数を数えます。

### 第1段階：ブロックを探す

ブロック数は、

\[
\frac{n}{m}
\]

です。

線形探索の平均は、おおよそ半分まで調べると考えるので、

\[
\frac{1}{2}\times\frac{n}{m}
=\frac{n}{2m}
\]

回です。

### 第2段階：ブロック内を探す

1ブロックには `m個` のデータがあります。

その中を線形探索するので、平均比較回数は、

\[
\frac{m}{2}
\]

回です。

### 2つを足す

したがって、

\[
\frac{n}{2m}+\frac{m}{2}
\]

となります。

```text
外側：n/m個の半分
→ n/(2m)

内側：m個の半分
→ m/2
```

## 公式を覚えずに解く方法

次の順番で式を作ります。

```text
1. 探索が何段階あるか確認する
2. 各段階の探索対象数を求める
3. 線形探索なら平均は約半分
4. 各段階の比較回数を足す
```

今回の形なら、

```text
ブロック数：n/m
ブロック内：m
```

なので、

```text
ブロック探索：n/(2m)
ブロック内探索：m/2
```

と復元できます。

この考え方は、式を忘れても使えます。

## よくある誤解・混同

### 誤解1：ブロック数は m 個である

1ブロックに `m個` 入るので、ブロック数は `m個` ではありません。

\[
\text{ブロック数}=\frac{n}{m}
\]

です。

### 誤解2：比較回数は最大値で考える

平均比較回数を問われているので、線形探索では約半分まで探すと考えます。

```text
最大比較回数
→ ほぼ全部

平均比較回数
→ ほぼ半分
```

### 誤解3：ブロックを探す比較だけでよい

ブロックを見つけただけでは、目的のデータ自体はまだ特定できません。

その後に、ブロック内の探索が必要です。

### 誤解4：二分探索と同じである

ブロック探索では、ブロック末尾の並びもブロック内も、基本的には線形探索として考えます。

二分探索のように、比較のたびに探索範囲を半分にする方法ではありません。

### 誤解5：+1を必ず付ける

線形探索の厳密な平均は、対象数が `N個` なら、

\[
\frac{N+1}{2}
\]

と表せます。

ただし、問題では十分大きな値を前提に定数項を無視したり、2段階の補正が相殺されたりすることがあります。

試験では、選択肢と問題文の条件を確認しつつ、まずは

```text
平均は約半分
```

で式の骨格を作るのが安全です。

## まとめ（試験直前用）

- ブロック探索は「ブロックを探す」「ブロック内を探す」の2段階
- ブロック数は `n/m`
- 線形探索の平均比較回数は約半分
- 第1段階は `n/(2m)`、第2段階は `m/2`
- 公式を忘れても「外側を半分、内側も半分」で復元できる
