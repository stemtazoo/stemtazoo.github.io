## まず結論

浮動小数点形式とは、**符号・指数部・仮数部を使って小数を表す形式**です。

基本情報技術者試験では、ビット列を細かく計算する前に、次の3つを見ます。

| 部分 | 役割 | 最大値を表すとき |
|---|---|---|
| S | 符号 | 正の数にするので0 |
| E | 指数部 | 値を大きくしたいので最大 |
| M | 仮数部 | 値を大きくしたいので最大 |

今回のように、値が

```text
(-1)^S × 16^(E-64) × 0.M
```

で表されるなら、最大値にするには次のようにします。

```text
S = 0
E = すべて1
M = すべて1
```

そして、できた2進数を4ビットずつ区切って16進数に直します。

## 直感的な説明

浮動小数点形式は、小数をそのまま並べるのではなく、**符号・大きさ・細かい値**に分けて表す方法です。

ざっくり言うと、次の3つに分けます。

```text
S：正か負か
E：どれくらい大きいか
M：細かい数値部分
```

最大値を作りたいなら、考え方はシンプルです。

```text
正の数にする
→ Sは0

指数を大きくする
→ Eは全部1

仮数を大きくする
→ Mは全部1
```

つまり、式を全部計算しなくても、**最大値なら、符号以外をできるだけ1にする** と考えればよいです。

## 定義・仕組み

浮動小数点形式では、数を次のような部品で表します。

| 部品 | 意味 | 例 |
|---|---|---|
| 符号部 | 正負を表す | 0なら正、1なら負 |
| 指数部 | 小数点の位置や値の大きさを表す | Eが大きいほど値が大きくなりやすい |
| 仮数部 | 有効数字の部分を表す | Mが大きいほど値が大きくなりやすい |

今回の形式では、24ビットを次のように分けています。

```text
S：1ビット
E：7ビット
M：16ビット
```

合計で24ビットです。

```text
S EEEEEEE MMMMMMMMMMMMMMMM
```

値は次の式で表されます。

```text
(-1)^S × 16^(E-64) × 0.M
```

ここで、`E-64` のように一定値を引いているのは、指数部で正の指数も負の指数も表せるようにするためです。

このテーマは、基本情報技術者試験の「データ表現」や「コンピュータシステム」と関係する内容です。公式の出題範囲やシラバスは、[IPA：基本情報技術者試験](https://www.ipa.go.jp/shiken/kubun/fe.html) から確認できます。

## 科目Aでどう出る？

科目Aでは、浮動小数点形式のビット列から、最大値や最小値を表すものを選ぶ問題が出ることがあります。

効率よく解く手順は、次のとおりです。

1. 符号部Sを確認する
2. 正の最大値ならSを0にする
3. 指数部Eを最大にする
4. 仮数部Mを最大にする
5. 4ビットずつ区切って16進数に直す

今回と同じ考え方で、24ビットを考えます。

```text
S = 0
E = 1111111
M = 1111111111111111
```

これをつなげると、次のビット列になります。

```text
0 1111111 1111111111111111
```

見やすく並べると、

```text
011111111111111111111111
```

これを4ビットずつ区切ります。

```text
0111 1111 1111 1111 1111 1111
```

16進数に直すと、

```text
7 F F F F F
```

なので、答えは次のようになります。

```text
7FFFFF
```

## 科目Bでどう使う？

科目Bで、浮動小数点形式のビット列を直接選ばせる問題は多くありません。

ただし、プログラムやアルゴリズムの問題で、数値の表現範囲、丸め誤差、桁落ち、オーバーフローのような話が出る可能性はあります。

そのときは、細かいビット列の暗記よりも、次の感覚が役立ちます。

```text
指数部
→ 表せる値の大きさの範囲に関係する

仮数部
→ 有効桁数や細かさに関係する
```

つまり、科目Bでは次のように位置づけます。

```text
ビット列から16進数を選ぶ問題
→ 科目A向き

数値誤差や表現範囲を読む問題
→ 科目Bでも補助知識として役立つ
```

## よくある誤解・混同

浮動小数点形式では、次の混同がよく起こります。

| 混同 | 正しい切り分け |
|---|---|
| 最大値なので全部1にすればよい | 符号部Sまで1にすると負の数になる |
| 先頭が1でも大きいと思う | 先頭が符号なら、1は負を表す |
| 指数部だけ最大にすればよい | 仮数部も最大にする必要がある |
| 仮数部だけ見ればよい | 指数部も値の大きさに大きく関係する |
| 2進数から16進数への変換で桁をずらす | 4ビットずつ区切って変換する |

特に、最大値を問われたときは、次の順で見ます。

```text
まず符号を見る
→ 正の最大値ならS=0

次に指数を見る
→ Eを最大

最後に仮数を見る
→ Mを最大
```

先頭ビットが符号部なら、`BFFFFF` や `FFFFFF` のように先頭ビットが1になる選択肢は、正の最大値ではありません。

## まとめ（試験直前用）

- 浮動小数点形式は、符号部・指数部・仮数部で小数を表す形式
- Sは符号、Eは指数部、Mは仮数部
- 正の最大値を作るなら、Sは0にする
- Eは値の大きさ、Mは細かい数値部分に関係する
- 最大値なら、EとMをすべて1にする
- 2進数を16進数に直すときは、4ビットずつ区切る
- 科目Aではビット列問題、科目Bでは数値誤差や表現範囲の補助知識として使う
