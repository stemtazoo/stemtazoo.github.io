## まず結論

ビジネスインパクト分析とは、**システム障害や災害で業務が止まったとき、事業にどれくらい影響が出るかを評価する分析**です。

基本情報技術者試験では、次のように切り分けると選択肢を選びやすくなります。

```text
影響を分析し、最大停止時間を決める → ビジネスインパクト分析
事業を続けるための計画そのもの     → BCP
継続的に運用・改善する活動         → BCM
復旧目標の時間                     → RTO
許容できる最大停止時間             → MTD
```

「情報システムに許容される最大停止時間を決定する」と出たら、**ビジネスインパクト分析** の実施事項と判断します。

## 直感的な説明

ビジネスインパクト分析は、ひとことで言うと **止まったらどれだけ困るかを調べること** です。

例えば、同じシステム障害でも、影響の大きさは業務によって違います。

```text
受注システムが止まる → 売上や顧客対応に大きく影響する
社内掲示板が止まる → 困るが、すぐに事業停止とは限らない
```

このように、どの業務が重要か、どれくらい止まると危険かを整理します。

その結果をもとに、BCPで

```text
どの業務を優先して復旧するか
どの時間までに復旧するか
どんな代替手段を用意するか
```

を考えます。

## 定義・仕組み

ビジネスインパクト分析は、BCPを作る前提として行われる分析です。

主に次のようなことを整理します。

| 観点 | 内容 |
|---|---|
| 重要業務 | 止まると事業への影響が大きい業務はどれか |
| 影響度 | 売上、顧客、法令、信用などにどの程度影響するか |
| MTD | 最大でどれくらい停止を許容できるか |
| RTO | どの時間までに復旧したいか |
| 復旧優先度 | どの業務・システムから復旧するか |

特に、試験で押さえたいのが **MTD** と **RTO** です。

```text
MTD：Maximum Tolerable Downtime
     許容できる最大停止時間

RTO：Recovery Time Objective
     目標復旧時間
```

MTDは「ここを超えると事業への影響が大きすぎる」という限界です。

RTOは「その限界より前に、ここまでには復旧したい」という目標です。

```text
MTD：最大で24時間までなら耐えられる
RTO：できれば8時間以内に復旧したい
```

のように考えます。

公式の出題範囲やシラバスは、[IPA：基本情報技術者試験](https://www.ipa.go.jp/shiken/kubun/fe.html) から確認できます。BCPやビジネスインパクト分析は、サービスマネジメントや事業継続の考え方と関係が深い内容です。

## どんな場面で使う？

ビジネスインパクト分析は、BCPを作る前や見直す前に使います。

例えば、次のような場面です。

- 地震や水害でシステムが止まった場合を想定する
- サーバ障害で業務が止まった場合の影響を整理する
- 重要業務とそうでない業務を分ける
- 復旧の優先順位を決める
- 最大停止時間や目標復旧時間を決める

ここで大事なのは、BIAは **計画書を作る作業そのもの** ではないことです。

BIAは、BCPを作るために、事前に影響を分析する作業です。

```text
BIAで影響を分析する
→ その結果をもとにBCPを作る
→ BCMとして訓練・教育・見直しを続ける
```

という流れで考えると分かりやすいです。

## よくある誤解・混同

一番混同しやすいのは、**BIA、BCP、BCMの役割** です。

| 用語 | 役割 |
|---|---|
| BIA | 事業への影響を分析する |
| BCP | 事業を継続・復旧するための計画 |
| BCM | BCPを含めて、教育・訓練・見直しまで行う継続的な管理活動 |

選択肢では、次のように切り分けます。

| 選択肢の内容 | 判断 |
|---|---|
| BCPの有効性をテストする | BCP策定後の検証・訓練 |
| 代替手順や復旧手順を教育する | BCM活動の一部 |
| 環境変化を踏まえてBCPを見直す | BCM活動の一部 |
| 許容される最大停止時間を決定する | BIAの実施事項 |

今回のように「ビジネスインパクト分析での実施事項」と聞かれたら、**教育・テスト・見直しではなく、影響評価や停止時間の決定** を選びます。

また、RTOとMTDも混同しやすいです。

```text
MTD：許容できる最大停止時間
RTO：復旧したい目標時間
```

MTDは限界、RTOは目標です。

## まとめ（試験直前用）

- ビジネスインパクト分析は、障害や災害による **事業への影響を評価する分析**
- BIAでは、重要業務、影響度、復旧優先度、MTD、RTOなどを整理する
- 「最大停止時間を決定する」はBIAの実施事項
- BCPは計画、BCMは教育・訓練・見直しを含む継続的な管理活動
- MTDは限界、RTOは復旧目標と切り分ける
