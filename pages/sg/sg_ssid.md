## まず結論

SSIDは、無線LANで接続先のネットワークを識別するための名前です。

スマートフォンやPCでWi-Fiを選ぶときに表示される、次のような名前がSSIDです。

- `Company-WiFi`
- `Guest-WiFi`
- `Home-5G`

SG試験では、SSIDそのものよりも、BSSID、MACアドレス、SSIDステルスとの違いが問われやすいです。

| 用語 | 判断ポイント |
|---|---|
| SSID | 無線LANのネットワーク名 |
| BSSID | アクセスポイントごとの識別情報 |
| SSIDステルス | SSIDを一覧に表示されにくくする設定 |

## 直感的な説明

SSIDは、無線LANに付ける「看板の名前」のようなものです。

たとえば、会社で社員用と来客用の無線LANを分ける場合、SSIDを分けることで利用者が接続先を選びやすくなります。

```text
社員用 : Company-WiFi
来客用 : Guest-WiFi
```

ただし、SSIDはあくまで接続先を識別する名前です。SSIDが分かることと、安全に接続できることは別です。

安全性は、WPA2/WPA3などの暗号化方式、認証方式、アクセス制御などと組み合わせて考えます。

## 定義・仕組み

SSIDは、Service Set Identifierの略です。

無線LANでは、アクセスポイントや端末にSSIDを設定し、端末はそのSSIDを見て接続先のネットワークを選びます。SSIDの長さは最大32オクテットです。

| 項目 | 内容 |
|---|---|
| 正式名称 | Service Set Identifier |
| 役割 | 無線LANのネットワークを識別する |
| 表示されやすい場所 | Wi-Fi接続先の一覧 |
| 混同しやすい用語 | BSSID、ESSID、MACアドレス、SSIDステルス |

同じSSIDを複数のアクセスポイントに設定することもあります。たとえば、広いオフィスや学校では、同じSSIDを使うことで移動中も接続を維持しやすくします。

この場合、利用者からは同じSSIDに見えますが、実際には接続先のアクセスポイントが切り替わることがあります。

## どんな場面で使う？

SSIDは、無線LANの接続先を選ぶ場面で使われます。

代表的には、次のような場面です。

- 社員用Wi-Fiと来客用Wi-Fiを分ける
- 2.4GHz帯と5GHz帯で接続先を分ける
- 店舗や拠点ごとに無線LAN名を設定する
- 複数アクセスポイントで同じネットワーク名を使う

SG試験では、SSIDを「セキュリティ対策そのもの」と考えないことが重要です。

たとえば、SSIDを変える、SSIDを隠す、SSIDを複雑にする、といった説明だけでは、強い認証や暗号化を行ったことにはなりません。

選択肢では、次のように切り分けます。

| 表現 | 判断 |
|---|---|
| 接続先として表示される無線LAN名 | SSID |
| アクセスポイントを識別する情報 | BSSID |
| SSIDを一覧に表示させにくくする | SSIDステルス |
| 通信内容を読まれにくくする | 暗号化方式 |

## よくある誤解・混同

### SSIDを隠せば安全になる

SSIDステルスを使うと、SSIDが一覧に表示されにくくなります。しかし、SSIDを完全に秘匿できるわけではありません。

SG試験では、「SSIDを隠すことは補助的な対策であり、本質的な安全対策ではない」と押さえます。安全性を高める主役は、適切な暗号化方式や認証方式です。

### SSIDはアクセスポイントのMACアドレスである

これはBSSIDとの混同です。

SSIDは無線LANのネットワーク名です。アクセスポイントの識別情報として問われる場合は、BSSIDを考えます。

### 同じSSIDなら同じアクセスポイントである

同じSSIDでも、アクセスポイントが1台とは限りません。

会社や学校では、複数のアクセスポイントに同じSSIDを設定することがあります。この場合、端末は同じSSIDの中で接続先アクセスポイントを切り替えながら通信することがあります。

### SSIDだけでアクセス制御できる

SSIDは接続先を識別する名前であり、利用者を認証する仕組みではありません。

「誰が接続できるか」を制御するには、認証方式、パスフレーズ、証明書、アクセス制御などが必要です。選択肢に認証や暗号化の説明がない場合は、SSIDだけで安全と言い切っていないかを確認します。

## まとめ（試験直前用）

- SSIDは、無線LANのネットワーク名です。
- BSSIDは、アクセスポイントごとの識別情報です。
- SSIDを隠すことは補助的対策であり、強い認証や暗号化の代わりにはなりません。
- 同じSSIDでも、複数のアクセスポイントが使われることがあります。
- 選択肢では「ネットワーク名」「接続先一覧」ならSSID、「アクセスポイント識別」ならBSSIDを疑います。
